# 建筑三档质量与加载合同

- Status: Adopted
- Decision date: 2026-07-25
- Scope: 新华路世界中的生产建筑资产、地图展示和本地游览加载
- Supersedes: 早期实验中“地图按距离展示 Massing，再切换 Identity”的规则，以及 2026-07-24 将“进入建筑”误作 Hero 开关的结论

## 目标

本合同同时解决两个问题：

1. 扩展建筑生产管线，使每栋生产建筑具备 Hero、Identity 和 Massing 三档资产；
2. 地图全览只承担轻量且可识别的 Identity；本地游览中，完整 Hero 仅由玩家到建筑的实际距离按需加载。

三档质量的正式关系为：

`Hero / Full → Hybrid Identity → Massing`

Massing 是内部技术占位，不是可交付给玩家观看的建筑缩略图。

## 第一档：Hero / Full

Hero 是建筑的最高质量版本。

- 使用 Blender 制作，并遵守 `docs/research/blender-ai-workflow.md`；
- 保留完整轮廓、立面、入口、材质、标志性构件和近景细节；
- 作为该建筑的最高质量母版本和本地近景游览资产；
- 通过 Blender、GLB 和真实 Three.js 页面三层验收；
- 记录可编辑 `.blend`、正式 `.glb`、生成器、build record 和运行时验收证据。

Hero 不在地图全览阶段批量加载。只有在本地游览中进入该建筑的近景距离，才按需加载。

## 第二档：Hybrid Identity

Identity 是正式的、长期可见的建筑精简版本。它的目标不是“有一个占位体块”，而是在较低成本下仍能让玩家辨认出具体建筑。

Identity 沿用上海影城旧版 Hybrid Identity 的制作方法：

- 规则主体由程序化几何构建；
- 窗格、立柱、楼梯、板缝等重复构件采用实例化、合批或共享材质；
- 丝带、异形屋顶、拱门、椭圆洞口等不可替代的独特轮廓保留为轻量 GLB；
- 不把完整 Blender Mesh 顶点简单改写进 JavaScript 冒充程序化优化；
- 不以牺牲建筑身份为代价追求零 GLB；
- 性能结果必须以实际体积、三角面、材质、节点和 draw calls 验证，不能假设程序化实现天然更快。

### Identity 可见质量合同

每栋 Identity 至少满足：

1. 保留至少三处该建筑独有的识别构件；
2. canonical 观察方向下一眼可识别；
3. 正面、侧面和背面均有合理建筑表达，不能出现裸露方盒或大面积无意义空白墙；
4. 保持 Hero 的位置、朝向、比例、地面基准和主要轮廓关系；
5. 在地图预期屏幕占比下仍能读出主体层级；
6. 允许省略门头小字、室内、场地小品和微小连接件，但不能省略决定建筑身份的轮廓；
7. Identity 加载失败时不能退回玩家可见的裸方盒。

上海影城旧版 Hybrid Identity 是该方法的当前参考基线：规则体块与重复构件程序化，白色连续丝带、椭圆开洞、玻璃鼓体和后塔楼等身份轮廓由轻量资产保留。

Identity 是一项独立的生产资产，需要独立生成、审计和验收；它不等同于 Hero 的自动减面结果。未来可以增加从 Hero 半自动导出 Identity 的工具，但自动化不能降低上述可见质量门槛。

## 第三档：Massing

Massing 只表达建筑的位置、占地、高度和大致体积。

- 可以使用最简单的程序化体块；
- 不承担建筑身份表达；
- 仅允许在启动封面完全遮挡期间或渲染系统内部短暂存在；
- 地图对玩家可见前必须已经替换为 Identity；
- 进入任何可交互模式后，不得出现一帧可见 Massing；
- Massing 不参与地图远近切换，也不作为弱网下的长期降级画面。

## 地图展示规则

地图和全览模式始终展示 Identity。

- 地图首次对玩家可见时，所有可见建筑至少已经达到 Identity；
- 不根据玩家距离、地图人物位置或镜头远近切换建筑质量；
- 拉近或远离地图镜头时，建筑保持同一 Identity；
- 地图不批量加载 Hero；
- 小地图、全览地图和地图缩放中的建筑缩影属于 Identity 的不同屏幕呈现，不是新的质量档位。

地图阶段的稳定状态为：

`Identity → Identity`

而不是：

`Massing → Identity → Full`

## 本地游览与快速定位规则

质量提升只发生在本地游览，并且只由玩家到每栋建筑的实际距离决定：

1. 玩家在全览中选择“进入”，只把人物和相机快速定位到对应地点；
2. `start`、建筑 ID、按钮选择或目的地 preset 都不得直接决定质量档位；
3. 本地游览持续计算玩家与所有建筑的实际平面距离；
4. 玩家进入建筑的 Hero 进入阈值时，保留 Identity 作为加载兜底并按需加载 Hero；
5. Hero 加载完成后在同一原点平滑替换 Identity；
6. 已显示 Hero 的建筑使用更远的退出阈值，避免在边界附近反复闪烁；
7. 玩家超过退出阈值后释放 Hero，继续显示 Identity；
8. 快速定位后与自然步行到达完全复用同一条距离判定路径。

本地游览的切换依据只有空间距离，不是“进入建筑”、目的地 preset 或详情状态。

标准网络下允许 Identity 与 Hero 距离切换；弱网长期保留 Identity，不请求 Hero。

## 碰撞与空间一致性

- Hero、Identity 和 Massing 使用同一建筑原点、朝向和单位；
- 三档资产的地面接触点必须一致；
- 视觉质量切换不得导致建筑位移、缩放或旋转跳变；
- 碰撞应与视觉质量解耦，优先使用稳定的独立碰撞结构；
- Identity 切换为 Hero 后，玩家不能被弹开、卡住或落入地面。

## 生产管线

每栋生产建筑的资产记录至少包含：

- Hero Blender 生成器与可编辑 `.blend`；
- Hero `.glb`；
- Hybrid Identity 生成器、程序化 recipe 或参数；
- Identity 轻量 `.glb`（如果该建筑需要独特轮廓资产）；
- Massing 参数；
- 三档共享的原点、比例、朝向和碰撞配置；
- Hero 与 Identity 的 build record；
- canonical、侧向、背向和实际地图运行截图；
- Hero / Identity / Massing 的体积、三角面、节点、材质、图片和 draw-call 记录；
- 三档资产与建筑 ID 的 manifest 映射；
- 本地游览 Hero 进入阈值、退出阈值和采样间隔。

普通建筑允许复用经过验收的 Hero 或 Identity 原型，但每个地图建筑实例仍必须明确映射到三档资产或参数，不能依赖临时方盒作为正式展示。

## 加载策略

- 启动封面期间可以建立 Massing 和基础世界状态；
- 封面退出前准备地图所需 Identity；
- 地图阶段不加载所有 Hero；
- “进入”只改变人物和相机位置；
- 本地游览按实际距离加载附近 Hero；
- Hero 加载完成前继续显示 Identity，不出现空白或方盒；
- 离开退出阈值后可以根据内存预算释放 Hero；
- 弱网与加载失败时长期保留 Identity；
- 缓存版本必须随实际 Identity 或 Hero 二进制变化。

## 验收要求

### Identity

- canonical、侧向和背向均不存在裸方盒；
- 至少三处身份构件可读；
- 在实际地图视口和目标屏幕占比下可识别；
- 弱网状态下可以长期作为最终可见画面；
- 资源预算、draw calls 和加载时间均有实测记录。

### Hero 切换

- 全览中无论距离多近都不触发；
- 本地游览只由实际距离触发，不读取进入目标或目的地 preset；
- 快速定位与自然步行到达得到相同结果；
- 加载期间 Identity 持续可见；
- 进入与退出阈值形成滞回，边界附近不反复闪烁；
- 切换无明显位置、比例、朝向和地面接触跳变；
- 碰撞稳定；
- 远离后恢复 Identity。

### 地图

- 地图首次可见时没有 Massing；
- 地图缩放和玩家移动不会触发质量档位变化；
- 不会因 Hero 加载失败而影响地图游览；
- 浏览器控制台没有新增错误；
- 弱网和标准网络均通过真实页面验收。

## 当前正式决策

1. 上海影城恢复使用旧版 Hybrid Identity 方案作为第二档参考，不采用裸露程序化方盒；
2. Hero 继续使用完整 Blender / GLB 资产；
3. Massing 保留为玩家不可见的内部占位；
4. 地图始终至少展示 Identity，不进行基于距离的建筑质量切换；
5. “进入建筑”只负责快速定位，不是 Hero 开关；
6. 本地游览中的 Hero / Identity 只按玩家到建筑的实际距离切换；
7. 当前只有上海影城的 Hybrid Identity 达到本合同参考标准，其他建筑仍是待迁移状态；
8. 后续建筑生产管线必须同时管理 Hero、Hybrid Identity 和 Massing。
