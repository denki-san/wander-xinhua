# 幸福里新版模型精细化管线可行性调研

## 结论

幸福里适合进入新版 Blender → GLB → Three.js 精细化管线，预期收益高。但不建议把现有 `XingfuliBlock` 整体替换为单一大 GLB；应采用“建筑硬表面进入 Blender，运行时街景与交互继续留在 React Three Fiber”的混合方案。

建议在当前并行模型改动收口并形成干净基线后，把幸福里作为街区型资产试点。第一轮只做中庭 30 至 40 米范围，通过后再扩展到东西入口。

## 当前状态（已观察）

- `app/scene/xingfuli-block.tsx` 是运行时主体，七栋体量来自 `app/scene/xingfuli-layout.json`，当前不是 GLB。
- 已有七栋建筑、入口绿墙、白色玻璃转角、倒影池、池中树、木桥、街灯、长椅、红伞、盆栽和精细碰撞。
- 已有三张同主体本地照片，并选择 `courtyard-canonical.jpg` 为中庭 canonical view。
- 现有细化主要以 JSX 构件层数为门槛：窗 3→6、店面 3→7、池沿 1→9。
- 当前缺少专用 Blender 生成器、`.blend`、`.glb`、固定机位 Blender 预览和 GLB 结构审计。
- 资料能可靠证明中庭与水景，但入口近景、建筑侧面、屋顶和背面覆盖不足。

## 公开资料补充

- 项目介绍确认其由原上海橡胶制品研究所更新而来，通过拆墙连接番禺路与幸福路，形成一字型开放步行主轴：[项目介绍](https://www.blissoffice.com.cn/index/Article/show/catid/5/id/475.html)。

## 新管线的主要收益

1. 把 box 叠层升级为真实构造深度：可控 bevel、窗洞凹入、窗框转角、雨棚厚度、阳台板边、栏杆节点、女儿墙压顶和檐口收口。
2. 让七栋楼形成不同身份：银灰悬挑盒体、不规则阳台、玻璃包角、木格栅、黑玻璃亭、西端抽象墙面等不再只是颜色差异。
3. 建立番禺路入口、中庭 canonical、反向巷道、水景侧向四个固定机位，用照片对照而不是 mesh 数量判断质量。
4. 保存可编辑 `.blend` 和确定性生成器，并增加 GLB 结构审计与真实页面验收。

## 推荐混合架构

### Blender / GLB

- 七栋建筑的体块、屋顶、窗门、阳台、悬挑、格栅和入口硬表面；
- 倒影池硬质池沿、桥体和固定石景；
- 按西段、中段、番禺路入口段拆成三个空间资产，保留视锥裁剪和分批加载能力；
- 建议生成器为 `scripts/create_xingfuli_models.py`，源文件放入 `assets/models/source/xingfuli/`，GLB 放入 `public/models/xingfuli/`。

### React Three Fiber

- 动态水面、喷泉、透明排序、昼夜灯光；
- 植物、街灯、座椅、桌椅、红伞和盆栽 instancing；
- 角色、行动点、现有碰撞、池边绕行、入口净空和性能分级；
- GLB 加载失败时的简化 fallback。

## 动工前仍需补齐（未知）

1. 番禺路入口正面与左右 45°；
2. 幸福路入口及西端墙面近景；
3. 七栋楼侧面分界和首层入口节奏；
4. 至少两张屋顶或高位视图；
5. 可用于尺度校准的门、栏杆、人物或铺地模块；
6. 2026 年现场是否已经更换店面、外摆、墙绘或绿化。

未知项不能用生成式补全冒充现场事实；证据不足的背面和屋顶应保持低细节、低对比。

## 首轮质量预算

| 项目 | 建议值 |
| --- | --- |
| 总三角面 | 三段合计 12 万至 18 万 |
| 文件体积 | 单段 2 至 4 MB，整体不超过 10 MB |
| 材质 | 共享 8 至 12 个；玻璃、发光、opaque 分离 |
| 图片贴图 | 不嵌入参考照片；首轮优先几何和纯材质 |
| 渲染节点 | 每段不超过 8 至 12 个 |
| 碰撞 | 沿用结构化简化碰撞，不从复杂网格自动生成 |

预算只是起点。若身份仍不清晰，应先改体块和专属构件，不继续细分通用窗框。

## 实施顺序与 Go / No-Go

1. 新增入口、侧面、屋顶证据，更新 manifest，并按模板写完整 Brief。
2. 只做中庭两侧四栋楼与倒影池的 massing pilot。
3. 同机位证明试件明显优于 JSX 版本，且没有破坏通行和移动端性能。
4. 补银灰悬挑、阳台、玻璃包角、木格栅、黑玻璃亭和入口绿墙关系。
5. 接回动态水面、池中树、外摆、instancing 和现有碰撞。
6. 完成 Blender、GLB、`/?start=xingfuli` 与 `/?start=hero` 三层验收。

如果入口和屋顶照片补不齐，可以只做中庭精细化；不应在证据不足时宣称全园区“高精度还原”。

## 最终判断

**可以细化，而且值得做。** 现有模型已经把空间骨架和玩法边界做对，新管线应把七栋楼从“程序化盒子 + 立面贴层”升级为可编辑、可固定机位对照、可审计的街区级硬表面资产，而不是重建地图或单纯堆面数。
