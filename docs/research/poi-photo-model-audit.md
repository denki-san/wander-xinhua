# POI 模型与真实照片审计

更新时间：2026-07-17
审计范围：全览地图 `MAP_POIS` 的 17 个 POI

## 结论

- **运行时模型覆盖：17/17。** 每个 POI 都会渲染程序化场景、独立 GLB 或复用 GLB。
- **可编辑来源覆盖：17/17。** 程序化模型保留 TSX，GLB 保留 Blend 与确定性生成脚本。
- **本地真实照片覆盖：17/17。** 每个 POI 已指定 1–3 张本地参考照片和一张典型对照图。
- **卡片照片对应：修复后 17/17。** 卡片现在只使用 `public/images/poi/` 内经过主体核对的本地实景。
- **真实模型一一对应：已覆盖 17/17。** 新华别墅 211 弄与 329 弄已拆成两个独立 GLB；211 弄以“外国弄堂”入口、门卫房和狭长车道为主体，329 弄以红瓦花园住宅、玻璃阳光房和庭院为主体。

## 下一步建模门槛

1. 只有清单内的本地照片可以进入建模 brief。
2. 每个 POI 最多选三张；选择多张时必须是不同视角，且来源证据必须能证明是同一建筑或同一园区。
3. 新模型必须与本轮基线比较：增加至少一倍可见专属构件，或三角面达到基线的两倍；不能用无意义细分充数。
4. 新华别墅 211 弄与 329 弄已拆成两个独立场所资产；旧的共享 GLB 仅为回退保留，不再被运行时引用。
5. 碰撞必须按实体构件分解，禁止使用超过建筑可见包络的单一大盒。
