# Messenger 复刻基准

## 产品边界

- 唯一视觉与交互参考：<https://messenger.abeto.co/>。
- 技术参考：<https://github.com/promptwhisper/messenger>。
- 微信长图只用于理解“新华路街区 / 一平米行动”的项目背景，不作为配色、版式、模型或交互参考。
- 产品是自由闲逛，不设置故事线、连续任务、打卡列表或地图看板。
- 全场只有一个可触发的行动点：“一平米行动”。

## 原站观察

固定桌面视口为 1440 × 900 后，原站初始体验具备这些高影响特征：

- 青绿色水彩天空占满视口，微型球形世界悬浮在中央。
- 场景、建筑和标题使用粗细不均的深色手绘轮廓，并叠加纸张颗粒感。
- “MESSENGER”标题像大型 3D 物件压在星球前方，黄色 BEGIN 按钮位于下方。
- 点击 BEGIN 后进入第三人称角色视角；角色可以行走、奔跑、跳跃，镜头跟随并允许拖拽旋转和滚轮缩放。
- 移动端提供触屏控制，而不是把桌面 UI 压缩成普通网页。
- 页面主体是 WebGL 场景，DOM 只保留轻量 HUD。

## 技术路线

- React Three Fiber + Three.js，延续当前 Vinext / React 19 项目。
- 球面移动采用“角色朝向法线 + 相机切平面方向”计算，不使用旧版二维 Canvas 坐标。
- 新华路场景使用代码生成的原创低多边形几何：里弄建筑、梧桐、路灯、自行车、长椅、包裹和一平米装置。
- 使用 toon ramp、屏幕空间轮廓和纸张颗粒后处理，重建原站的手绘观感。
- 桌面支持 WASD / Shift / Space / 拖拽 / 滚轮；手机支持摇杆和跳跃按钮。

## 验收条件

- 首屏不是二维地图，必须能看见真实 WebGL 3D 小世界。
- 可自由移动、奔跑、跳跃和控制镜头。
- 桌面 1440 × 900 与手机 390 × 844 均可用。
- 只出现一个行动点，不出现故事、三段任务或五个地点导航。
- 控制台无致命错误，WebGL 画面非空，生产构建通过。
- 最终产物不包含原站第三方资产，也不向原站发出运行时请求。
