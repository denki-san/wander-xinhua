# Blender、Three.js 与 AI 建模实用方法

## 结论

AI 操控 Blender 的强项不是一次性生成完整场景，而是把研究、生成、检查和返工
组织成可复现流水线。生产默认走确定性 Blender Python 与 Headless Blender；
Blender MCP 用于读取场景、调整相机、检查材质和局部视觉迭代。有价值的 MCP
修改必须回写生成器或版本化参数。

## 最短可用流程

1. 先保存真实参考图、来源元数据和 canonical comparison view。
2. 写模型 Brief：轮廓、比例、至少三个身份构件、材质、场地、预算和验收入口。
3. 按体块、身份构件、材质、场地、碰撞、优化分批生成；禁止一口气做完整场景。
4. 每批重建 `.blend`、导出 `.glb`、渲染 `test_` 预览并对照参考图。
5. 审计 GLB 的根变换、节点、三角面、材质、贴图政策、包围盒和动画。
6. 在真实 Three.js `?start=` 入口检查位置、比例、朝向、落地、碰撞、遮挡、
   控制台、加载和帧率。

## 质量门

- Blender 通过不等于完成；必须同时通过 Blender、GLB、Three.js 三层验收。
- 每个命名地标至少有三个主体独有构件，不能只换墙色和屋顶色。
- 参考照片只作研究证据，不嵌入 GLB，不作为运行时贴图。
- 优先让形体正确，再合并静态节点、共享材质和实例化重复资产。
- 渲染检查不合格就返工，不能把问题推迟到最终场景。

## 调研案例得到的方法

小红书视频《GPT 5.6 Sol 操控 Blender 有多强？》
（内容 ID：`6a546bdc0000000011019f0e`）展示了三类可迁移路径：

- 参考物研究后，在 Three.js 中构建可游览的博物馆；
- 将长卷画面拆成空间层次，再转成可浏览 3D 场景；
- 用 Blender 制作玩具和骨骼动画，再交给 Three.js 形成互动剧场。

共同方法是：先定义质量合同和研究任务，拆分资产批次，逐批渲染检查，保留时间与
质量预算，最后在 WebGL 运行时验收。项目执行细则见
`docs/research/blender-ai-workflow.md` 和
`docs/research/content-research-wiki-workflow.md`。
